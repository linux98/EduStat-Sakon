คู่มือปฏิบัติการผู้ดูแลระบบฉบับสมบูรณ์

คู่มือการควบคุม บริหารจัดการ และวิเคราะห์สารสนเทศทางการศึกษา จังหวัดสกลนคร (EduData Sakon) ทุกซอกทุกมุมสำหรับศึกษาธิการจังหวัด

dns 1. ภาพรวมระบบและสถาปัตยกรรมข้อมูล

ระบบ EduData Sakon พัฒนาขึ้นบนเทคโนโลยี Serverless ของ Google Cloud & Workspace (Google Apps Script) ทำงานร่วมกับ Google Sheets ในลักษณะของ Relational Table Database มีข้อดีคือปลอดภัยสูง เสถียร และค่าใช้จ่ายคลาวด์เป็นศูนย์ตลอดอายุการใช้งาน

ผังการหมุนเวียนข้อมูลและการทำงานของสถาปัตยกรรม (Data Flow Diagram)

รายละเอียดตารางข้อมูลใน Google Sheets

โครงสร้างความสัมพันธ์ของตารางฐานข้อมูลออกแบบเฉพาะตัวเพื่อสอดรับการนำเข้าข้อมูลสถิติจาก 20 สังกัด:

ชื่อตาราง / ชีต ฟิลด์ข้อมูลหลัก บทบาทและการเชื่อมโยง
Users username, password (SHA-256), name, role, agencyId, status เก็บรายชื่อบัญชีสิทธิ์และการเข้าสู่ระบบ โดยมีการ Hash รหัสผ่านปลอดภัย
Settings key, value ตั้งค่าปีการศึกษาปัจจุบัน และรอบการบันทึกข้อมูลหลัก
FormTemplates formId, title, agencyId, deadline, formConfig (JSON Schema) เก็บแบบฟอร์มที่แอดมินสร้างขึ้น โดยบันทึกโครงสร้างฟิลด์ในรูปแบบ JSON
Data dataId, formId, agencyId, username, status, formData (JSON String) เก็บผลสถิติคำตอบของแบบฟอร์มเป็น Dynamic JSON String รองรับคำถามหลายประเภท
AuditLogs logId, timestamp, username, action, details, ipAddress เก็บบันทึกประวัติกิจกรรมทุกขั้นตอนเพื่อใช้ตรวจสอบย้อนหลัง
GisCoords agencyId, lat, lng เก็บพิกัด Latitude, Longitude ของสำนักงานสังกัดสำหรับแผนที่ GIS

security 2. ความปลอดภัยและการจัดการเซสชัน

ความมั่นคงปลอดภัยของข้อมูลสารสนเทศจัดเป็นสิ่งสำคัญสูงสุด ระบบจึงแบ่งบทบาทสิทธิ์ (Roles) และกลไกควบคุมเซสชันของระบบอย่างเป็นสากล ดังนี้:

2.1 การแบ่งระดับสิทธิ์ผู้ใช้งาน (Access Control List)

  • Super Admin / Admin (ศึกษาธิการจังหวัด): สิทธิ์สูงสุด สามารถจัดการผู้ใช้, ตรวจสอบ Audit logs, พิมพ์รายงานของทุกสังกัด, ตีกลับข้อมูล และออกแบบแบบฟอร์มใหม่
  • Agency Admin (สพป./สพม./ท้องถิ่น): มองเห็นรายงานและสามารถกำกับติดตามเฉพาะสถานศึกษาภายใต้การปกครองของสังกัดตนเองเท่านั้น
  • School User (ผู้รายงานระดับโรงเรียน): กรอกข้อมูล บันทึกแบบร่าง ดึงเทมเพลต CSV และนำเข้าข้อมูลของโรงเรียนตนเอง
lock

การป้องกันการบันทึกทับซ้อน (Concurrency Control Lock)

หลังบ้านใช้กลไก LockService.getScriptLock() กำหนดคิวการเข้าบันทึกฐานข้อมูลชีต เมื่อมีคนส่งข้อมูลพร้อมกันในช่วงวันเดดไลน์ ระบบจะต่อคิวทำงานป้องกันไม่ให้ฐานข้อมูลเสียหายจากการชนกันของสคริปต์ (Data Corruption Prevention)

2.2 ระบบต่อเซสชันอัตโนมัติ (Session Heartbeat)

เนื่องจาก CacheService มีระยะเวลาหมดอายุของ Session Token สูงสุด 2 ชั่วโมงเพื่อป้องกันภัยไซเบอร์ แต่เพื่อไม่ให้ผู้ดูแลระบบที่กำลังป้อนคำถามหรือดูข้อมูลถูก Logout ออกอัตโนมัติ ระบบได้ฝัง **Session Heartbeat** ไว้ในส่วน Frontend

สคริปต์จะคอยรันคำสั่ง refreshSession คืนไปยังเซิร์ฟเวอร์แบบเบื้องหลัง (Background Fetch) ทุกๆ 25 นาที เพื่อขยายวันหมดอายุของ Cache (TTL Renew) ให้ทำงานได้ต่อเนื่อง 24 ชั่วโมงจนกว่าผู้ใช้งานจะกดคลิก "ออกจากระบบ" ด้วยตนเอง

radar 3. แดชบอร์ดควบคุมสถานการณ์ (Live "War Room")

หน้า Dashboard ออกแบบมาในลักษณะศูนย์วิเคราะห์ข้อมูลที่ตอบสนองโต้ตอบได้สมบูรณ์ (Interactive War Room)

3.1 การ์ดตัวชี้วัดหลักจังหวัด (KPI Cards Counter Animation)

ด้านบนสุดจะแสดงยอดรวมสถานศึกษา นักเรียน ครู และผู้เรียนต้องการพิเศษของทั้งสกลนคร เมื่อล็อกอินเข้ามาระบบจะเรียกฟังก์ชัน animateValue() เพื่อแสดงผลตัวเลขด้วยแอนิเมชันเลื่อนขึ้นอย่างรวดเร็ว (Animated Count Up)

3.2 แผงบอร์ดนำส่งรายงาน (Submission Leaderboard)

แสดงตารางความคืบหน้าการส่งข้อมูลของ 13 สังกัด เรียงลำดับตามความครบถ้วนในการอนุมัติข้อมูลแบบเรียลไทม์ พร้อมสัญลักษณ์เหรียญทอง เงิน ทองแดง กระตุ้นความร่วมมือ

ขั้นตอนการโต้ตอบสองทิศทาง (Bi-directional GIS & Dashboard Interaction)

จากแผนที่สู่แดชบอร์ด: เมื่อคลิกปุ่มมาร์กเกอร์บนแผนที่ GIS ของสังกัดใดๆ แล้วกดปุ่ม "แสดงข้อมูลสังกัดนี้" ตัวฟิลเตอร์สังกัด Dropdown ด้านบนจะเปลี่ยนอัตโนมัติ และสถิติทั่วหน้าจอจะเปลี่ยนตามโดยไม่ต้องรีเฟรชหน้าเว็บ

mail 4. ระบบกำกับติดตามและหน่วยทวงถาม AI

แก้ไขความยุ่งยากของแอดมินจังหวัดที่ไม่ต้องเปิดตารางเปรียบเทียบใน Excel และกดส่งอีเมลร่างแยกด้วยตนเองอีกต่อไป

4.1 ตารางสถานภาพความคืบหน้า (Matrix Grid View)

ในเมนู **กำกับติดตามรายงาน** แสดงตาราง Grid แสดงสถานะการส่งข้อมูลของทุกฟอร์มและทุกสังกัดในแบบรหัสสี:

  • 🟢 สีเขียว: อนุมัติข้อมูลแล้ว (Approved)
  • 🔵 สีฟ้า: ส่งข้อมูลมารอการตรวจสอบ (Pending)
  • 🟠 สีส้ม: ตีกลับข้อมูลให้แก้ไข (Rejected)
  • 🔴 สีแดง: ค้างส่งข้อมูลหรือค้างแบบร่าง (Missing / Draft)

4.2 คู่มือการใช้งาน "ทวงถามงานด้วย AI" (One-Click AI Dunning)

tips_and_updates

แสดงขั้นตอน: เลือกสังกัดค้างส่ง -> คลิก ✨ ทวงงานด้วย AI -> ตรวจร่างอีเมล -> กดส่งจดหมายทวงถาม

  1. เข้าเมนู กำกับติดตามรายงาน มองหาแถวสังกัดที่แสดงปุ่มสถานะสีแดงค้างส่ง
  2. คลิกปุ่มสีม่วงพิเศษ ✨ "ทวงงานด้วย AI" ทางช่องควบคุมของแถวนั้นๆ
  3. ระบบหลังบ้านจะติดต่อ Gemini AI พร้อมส่งชื่อเจ้าหน้าที่ รายการฟอร์มค้างส่ง และเดดไลน์ที่ใกล้ถึง
  4. AI จะทำการร่างจดหมายราชการภาษาไทยที่เหมาะสม สุภาพเรียบร้อย แต่แสดงความจำเป็นเร่งด่วน
  5. กล่องป๊อปอัปจะแสดงร่างจดหมาย แอดมินสามารถคลิกพิมพ์แก้ไข หรือแก้ไขรายชื่ออีเมลผู้รับได้ด้วยตนเอง
  6. คลิก "ส่งอีเมลทวงถามข้อมูล" ระบบจะรันคำสั่ง MailApp.sendEmail() เพื่อส่ง HTML อีเมลตกแต่งเรียบหรูตรงไปยังกล่องจดหมายผู้รับทันที พร้อมสร้างประวัติบันทึกการส่ง (Audit Logs) เป็นหลักฐานทางราชการ

edit_note 5. จัดการแบบฟอร์มและ CSV แม่แบบอัจฉริยะ

ระบบ Form Builder ช่วยให้ผู้ดูแลสามารถกำหนดขอบเขตและหัวข้อสถิติที่เปลี่ยนไปในแต่ละปีการศึกษาได้อย่างยืดหยุ่น

5.1 ระบบสร้างฟอร์มอย่างง่าย (Visual Form Wizard)

  • เข้าเมนู ศูนย์ควบคุม (Admin Console) ➡️ คลิกสร้างแบบฟอร์มใหม่
  • กรอกหัวข้อแบบฟอร์ม คำอธิบาย และกำหนดวันปิดรับ (Deadline)
  • เลือกสังกัดที่ต้องการจัดเก็บข้อมูล (สามารถเลือกเฉพาะบางสังกัดเพื่อรายงาน หรือเลือก "ทุกสังกัด")
  • กดปุ่ม "เพิ่มฟิลด์คำถาม" เพื่อกำหนดชนิดตัวแปร เช่น ตัวเลข (Number) สำหรับใช้บวกเลขนักเรียน หรือ ตัวเลือกสั้น (Dropdown)
  • กดปุ่ม "บันทึกแบบฟอร์ม" เพื่อประกาศใช้แบบฟอร์มเข้าสู่ระบบโดยตรง

5.2 ระบบเทมเพลต CSV แบบแก้ไขภาษาต่างดาว (Dynamic CSV Template Engine)

หนึ่งในจุดอ่อนของไฟล์ CSV ทั่วไปเมื่อเปิดใน Microsoft Excel ภาษาไทยคือจะแสดงผลภาษาเพี้ยนอ่านไม่ออก ระบบของ EduData Sakon ได้แก้ไขปัญหานี้อย่างถาวร:

cloud_download

การฝัง UTF-8 BOM ในไฟล์ CSV เทมเพลต

ระบบสร้างไฟล์ดาวน์โหลดตามฟิลด์แบบฟอร์มที่สร้างขึ้นโดยฝังรหัสพิเศษ Byte Order Mark (BOM - \ufeff) ไว้ที่ต้นไฟล์ เมื่อนำไฟล์เทมเพลตนี้ไปดับเบิลคลิกเปิดใน Excel ภาษาไทย จะแสดงภาษาไทยที่สมบูรณ์แบบทันทีโดยที่เจ้าหน้าที่สถานศึกษาปลายทางไม่ต้องตั้งค่าการนำเข้า (Import Text) ใดๆ เพิ่มเติม

6.2 ขั้นตอนการทำงานเบื้องหลังและการดึง JSON Widget

  2. ระบบทำการรวบรวมคำอธิบายตารางสถิติตั้งค่าและโครงสร้างคอลัมน์ข้อมูลนำส่งทั้งหมดของ EduData ส่งไปยังโมเดล Gemini 1.5
  3. AI วิเคราะห์หาประเภทกราฟที่คุ้มค่า คัดกรองแกน X และแกน Y ตามข้อมูลจริง และเขียนตอบกลับเป็น **JSON Configuration Schema**
  4. ระบบ JavaScript ฝั่ง Frontend อ่านค่า JSON และทำการอิมพอร์ตวาด Widget ชาร์ตสวยงามลงบนหน้าจอ Grid ของผู้ใช้อัตโนมัติโดยที่ผู้ใช้ไม่ต้องเขียนสูตรเลือกแกนหรือลากคอลัมน์เอง
  5. ผู้ใช้สามารถย้ายขนาด ดึงลาก ปรับความกว้าง (Drag & Drop Resize) และสั่งบันทึกแดชบอร์ดสัญจรส่วนตัวเก็บลง LocalStorage ได้ทันที

map 8. ชั้นข้อมูล GIS Overlays และระบบช่วยปักหมุดพิกัด

เพิ่มพลังวิเคราะห์สถิติผ่านการจำลองแผนที่เชิงพื้นที่ และยกระดับการแก้ไขปัญหาพิกัดไม่ครบถ้วนให้แอดมินทำงานง่ายสุดๆ

8.1 แผงจำลองการวิเคราะห์ 3 ชั้น (Map Overlays)

ระบบแผนที่บนแดชบอร์ดหลักรองรับการเปลี่ยนชั้นข้อมูลแผนที่ (Layer Select) ผ่านเมนูมุมขวาบน:

  1. 📍 พิกัดสำนักงานสังกัด: แสดงจุดพินปกติตามพิกัดที่ตั้งสังกัด โดยแสดงผลแยกแยะตามการรายงานข้อมูล
  2. 🟣 ความหนาแน่นผู้เรียน (Student Density Bubble Map): แสดงจุดวงกลมสีรุ้งขนาดตามสัดส่วนผู้เรียน ช่วยสะท้อนภาพเชิงพื้นที่ว่าอำเภอใดมีประชากรผู้เรียนกระจุกตัวอยู่มากที่สุด
  3. 🚨 สัดส่วนนักเรียนต่อครู (Shortage Overlay): ประเมินสัดส่วนภาระงานครูในพื้นที่สกลนคร:
    • 🔴 สีแดง (ครู 1 คน : นักเรียน > 25 คน): ขาดแคลนกำลังครูวิกฤต
    • 🟡 สีเหลือง (ครู 1 คน : นักเรียน 15 - 25 คน): ปานกลางตามมาตรฐาน
    • 🟢 สีเขียว (ครู 1 คน : นักเรียน < 15 คน): ครูดูแลนักเรียนได้อย่างใกล้ชิด/เพียงพอ

มีแผง Dynamic Legend ปรากฏขึ้นอธิบายเกณฑ์วิเคราะห์โดยเปลี่ยนประเภทเกณฑ์ล่างขวาให้ตรงกับชั้นข้อมูลนั้นโดยอัตโนมัติ

admin_panel_settings 9. ศูนย์ควบคุมความปลอดภัยระบบสูงสุด (Admin Console)

หน้าเมนู ศูนย์ควบคุมระบบ (Admin Console) ออกแบบมาสำหรับการจัดการสิทธิผู้ใช้งาน ข้อมูลอ้างอิง และดูแลความสงบเรียบร้อยหลังบ้าน

9.1 ระบบจัดการผู้ใช้งาน (User Control Panel)

แอดมินสามารถเพิ่มรายชื่อพนักงานสังกัด หรือโรงเรียนเข้าระบบได้โดยตรง:

  • การจัดการรหัสผ่าน: รหัสผ่านที่สร้างขึ้นจะผ่านฟังก์ชัน SHA-256 Hashing ก่อนถูกเขียนทับลงบน Google Sheets database เพื่อความมั่นคงปลอดภัยสูงสุด
  • การระงับการเข้าใช้ชั่วคราว: หากพบความเสี่ยงหรือพนักงานย้ายสังกัด สามารถคลิกปิดสถานะการทำงาน (สลับปุ่มเป็น Inactive) เพื่อปิดสิทธิ์บัญชีผู้ใช้ชั่วคราวได้ทันทีโดยไม่ต้องทำการลบข้อมูลประวัติบัญชี

9.2 การวิเคราะห์ประวัติระบบ (Audit Logs Screen)

ตรวจสอบประวัติการรายงานข้อมูลย้อนหลัง บันทึกการนำส่ง หรือค้นหาว่าแอดมินคนใดกดยกเลิกแบบฟอร์มเพื่อติดตามความสงบเรียบร้อยและความโปร่งใสในการนำเข้าข้อมูล

9.3 การขึ้นรอบปีการศึกษาใหม่ (New Term Reset Cycle)

เมื่อเข้าสู่รอบการจัดเก็บสถิติปีการศึกษาใหม่ แอดมินสามารถสลับปีการศึกษาในการตั้งค่า และกดคำสั่งล้างปีการศึกษาเพื่อล้างค่าแบบร่างและตั้งระบบเตรียมรับรายงานชุดใหม่ โดยที่ข้อมูลสถิติของปีการศึกษาเดิมจะถูกเก็บรักษาไว้อย่างครบถ้วนสำหรับใช้ในแผนภูมิ YoY (Year-over-Year) เพื่อดูแนวโน้มการเติบโตต่อไป
