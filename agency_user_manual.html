เรียนรู้วิธีการส่งรายงานสถิติแบบกรอกข้อมูลด้วยตนเอง การนำเข้าข้อมูลชุดใหญ่ผ่านไฟล์ Excel/CSV และการวิเคราะห์แดชบอร์ดเชิงพื้นที่และการโต้ตอบกับ AI วิเคราะห์ข้อมูล เริ่มสอนตั้งแต่ศูนย์แบบเข้าใจง่ายที่สุด

help_outline 1. แนะนำระบบสารสนเทศ EduData Sakon & ข้อมูลเริ่มจากศูนย์

ระบบ EduData Sakon เป็นระบบที่พัฒนาขึ้นสำหรับจัดเก็บบันทึกข้อมูลสถิติเชิงลึกด้านการศึกษาของจังหวัดสกลนคร เพื่อนำมาใช้ประกอบการสร้างแดชบอร์ดความก้าวหน้าและการวิเคราะห์ข้อมูลเชิงพื้นที่สำหรับผู้บริหารระดับสูง (BI Dashboard & GIS Spatial Map) เพื่อให้ได้ข้อมูลที่แม่นยำ รวดเร็ว และนำไปปฏิบัติได้จริง

บทบาทหลักของเจ้าหน้าที่สังกัด (Agency User Role): ทำหน้าที่รายงานผลข้อมูลดิบด้านการศึกษา (เช่น จำนวนเด็กหลุดนอกระบบ, จำนวนครูและทรัพยากร, หรือสถิติรายปี) ตามแบบรายงานที่ผู้ดูแลระบบส่วนกลาง (Admin) มอบหมายให้เสร็จสิ้นภายในวันกำหนดส่ง (Deadline)

สถานภาพของข้อมูลรายงาน (Data Submission States)

ในการรายงานข้อมูล ข้อมูลที่นำส่งจะมีสถานภาพที่แตกต่างกัน 4 สถานะสำคัญ ซึ่งบ่งบอกสิทธิ์การแก้ไขและการนำไปใช้ดังนี้:

สัญลักษณ์สถานะ ความหมาย การแก้ไขข้อมูล การคำนวณแดชบอร์ด
แบบร่าง (Draft) ข้อมูลถูกบันทึกไว้ในฐานข้อมูลคลาวด์แล้ว แต่ยังไม่ได้ส่งตรวจสอบอย่างเป็นทางการ ✓ แก้ไขได้ตลอดเวลา ✗ ไม่นำไปคำนวณ
รอตรวจสอบ (Pending) ข้อมูลถูกส่งเข้าคิวให้ Admin กลางจังหวัดตรวจสอบความถูกต้องเรียบร้อยแล้ว ✗ ล็อกห้ามแก้ไข ✗ ไม่นำไปคำนวณ
อนุมัติแล้ว (Approved) ข้อมูลถูกต้องสมบูรณ์ Admin ตรวจรับเรียบร้อยแล้ว ✗ ล็อกห้ามแก้ไข ✓ นำเข้าระบบวิเคราะห์ทันที
ตีกลับแก้ไข (Rejected) ข้อมูลมีจุดผิดพลาด หรือไม่ผ่านเกณฑ์ตรวจสอบ Admin ส่งกลับมาแก้ไข ✓ แก้ไขและส่งใหม่ได้ ✗ ไม่นำไปคำนวณ

vpn_key 2. การลงชื่อเข้าใช้งานและการขยายเซสชันอัตโนมัติ (Heartbeat System)

คุณจะได้รับบัญชีเข้าใช้งานระบบ (Username และ Password) จากแอดมินจังหวัดสกลนครเพื่อเริ่มต้นเข้าปฏิบัติงานในระบบ

login ขั้นตอนการล็อกอินเข้าสู่ระบบ
  1. เปิดลิงก์แอปพลิเคชันระบบสารสนเทศส่วนกลาง
  2. กรอก **Username** (ชื่อบัญชีผู้ใช้) และ **Password** (รหัสผ่าน)
  3. คลิกปุ่ม **"เข้าสู่ระบบ"** เพื่อรับสิทธิ์การนำส่งข้อมูล
favorite ระบบต่อเวลาอัตโนมัติ (Heartbeat)

เพื่อความปลอดภัยทางราชการ เซสชันเข้าใช้ของคุณจะถูกจำกัดเวลาทำงานสูงสุด **2 ชั่วโมง** ทว่าระบบเบื้องหลังมีกลไก **Session Heartbeat** คอยทำงานเงียบๆ ทุกๆ 25 นาที โดยจะดึงข้อมูลต่ออายุให้อัตโนมัติหากหน้าจอยังถูกเปิดไว้ ช่วยให้ทำงานค้างได้โดยไม่ต้องล็อกอินใหม่เรื่อยๆ

warning
ข้อพึงระวังกรณีลืมรหัสผ่าน: เนื่องจากรหัสผ่านของคุณถูกแฮชเป็นระบบรักษาความปลอดภัยแบบทางเดียว (SHA-256 Hash) บนฐานข้อมูลหลังบ้าน ทำให้ไม่มีใครสามารถอ่านรหัสผ่านจริงของคุณได้ หากคุณลืมรหัสผ่าน จะต้องติดต่อผู้ดูแลระบบ (Super Admin) จังหวัด เพื่อทำการส่งคำสั่งล้างระบบรหัสผ่านและสร้างรหัสผ่านใหม่ (Reset Password) เท่านั้น

edit_note 3. ขั้นตอนการกรอกและแก้ไขข้อมูลผ่านหน้าเว็บบราวเซอร์

วิธีที่ง่ายที่สุดในการรายงานข้อมูลคือการกรอกทีละสังกัดผ่านแบบฟอร์มคำถามบนเว็บบราวเซอร์โดยตรง

ขั้นตอนการกรอกและนำส่งข้อมูลแบบละเอียด:

1. การเข้าแบบฟอร์มและตรวจสอบกำหนดส่ง
  • คลิกที่เมนู **"รายงานข้อมูล" (Data Entry)** ทางแถบนำทางด้านซ้าย
  • ระบบจะแสดงรายการแบบฟอร์มรายงานที่สังกัดของคุณต้องส่ง
  • สังเกตสีสัญลักษณ์แสดงระยะเวลากำหนดส่ง (Deadline): สีแดงเด่นชัดบ่งบอกว่าเหลือเวลาไม่ถึง 7 วันในการจัดส่งด่วน
2. การกรอกข้อมูลและการดักจับข้อผิดพลาด
  • กรอกตัวเลขหรือข้อความลงตามช่องคำถามที่แสดงผลขึ้นมา
  • ระบบมีฟังก์ชันดักตรวจสอบค่าผิดพลาด (Validation) ทันที: เช่น หากช่องบังคับใส่จำนวนตัวเลข แต่นำตัวอักษรไปกรอก ระบบจะขึ้นเส้นสีแดงเตือนและปิดไม่ให้กดนำส่งเพื่อกันข้อมูลบิดเบี้ยว
3. ปุ่มบันทึกแบบร่าง (Draft) กับ ปุ่มนำส่ง (Submit)
  • **ปุ่มบันทึกแบบร่าง (Draft):** ใช้คลิกเมื่อยังกรอกข้อมูลไม่ครบถ้วน หรือต้องการกลับมาแก้ไขต่อ ข้อมูลจะถูกจัดเก็บไว้บนเซิร์ฟเวอร์อย่างปลอดภัยโดยยังไม่ส่งถึงมือแอดมิน
  • **ปุ่มนำส่งข้อมูล (Submit):** ใช้คลิกเมื่อตรวจสอบแน่ชัดว่าถูกต้อง 100% แล้ว ข้อมูลจะกลายเป็นสถานะรอตรวจ (Pending) และถูกล็อกทันทีเพื่อป้องกันการแก้ไขข้อมูลลับหลัง
4. การจัดการกรณีข้อมูลถูกส่งกลับแก้ไข (Rejected)
  • หาก Admin จังหวัดตรวจพบจุดผิดพลาดและทำการส่งกลับเพื่อแก้ไข สถานะรายงานจะแสดงเป็นสีแดง **"ตีกลับแก้ไข (Rejected)"**
  • ให้คลิกปุ่มแก้ไข คุณจะพบ **"ข้อเสนอแนะ/บันทึกช่วยจาก Admin"** เพื่อชี้จุดที่ต้องเปลี่ยน
  • แก้ไขข้อมูลตามประเด็นนั้นๆ และกดปุ่ม **"นำส่งข้อมูล"** อีกครั้งหนึ่ง

table_view 4. ขั้นตอนการจัดเตรียมและนำเข้าข้อมูลชุดใหญ่ผ่าน Excel / CSV

หากคุณมีข้อมูลโรงเรียนย่อยในสังกัดเป็นจำนวนมาก การคีย์ลงหน้าเว็บบอร์ดทีละรายการอาจใช้เวลานาน ระบบจึงรองรับการดาวน์โหลดเทมเพลต Excel กรอกข้อมูลง่าย และอัปโหลดนำเข้าพร้อมกันทันที

system_update_alt ขั้นตอนการจัดส่งด้วย Excel / CSV ทีละขั้น:
  1. เข้าเมนู **"รายงานข้อมูล"** และเลือกหัวข้อแบบรายงานที่ท่านต้องการจัดส่ง
  2. คลิกปุ่ม 📥 **"ดาวน์โหลดแม่แบบ CSV (Template)"** ด้านขวาบนของแบบฟอร์ม
  3. เปิดไฟล์แม่แบบที่ได้รับด้วยโปรแกรม **Microsoft Excel** (ระบบแนบสัญลักษณ์ UTF-8 BOM ป้องกันปัญหาภาษาไทยแตกกลายเป็นต่างดาวโดยอัตโนมัติแล้ว)
  4. กรอกสถิติตามแถวคอลัมน์โรงเรียนย่อยที่กำหนดให้ครบถ้วน
  5. **กฎเหล็ก:** ห้ามลบ ห้ามแก้ไขตัวสะกดคอลัมน์หัวตาราง (Headers) ลำดับแรกอย่างเด็ดขาด มิฉะนั้นระบบนำเข้าจะปฏิเสธการอ่านไฟล์
  6. เซฟบันทึกเอกสารให้อยู่ในฟอร์แมต **CSV (Comma Separated) UTF-8**
  7. กลับมาที่หน้าเว็บ กดปุ่ม 📤 **"นำเข้าข้อมูล CSV"** เลือกไฟล์ที่เพิ่งเซฟบันทึกขึ้นมา
  8. หน้าจอจะแสดง **"ข้อมูลพรีวิวก่อนจัดส่ง"** ให้คุณเช็คความเรียบร้อย หากพบฟิลด์สีแดงแสดงว่าข้อมูลไม่ได้ฟอร์แมต ให้กดแก้ไขใน Excel แล้วอัปโหลดใหม่
  9. เมื่อพร้อมแล้ว กดปุ่ม **"ยืนยันและนำส่งรายงาน"** เพื่อเปลี่ยนสถานะเป็น Pending
warning
ทำไมนำเข้า CSV แล้วพบเส้นสีแดงแสดง Error: ระบบจะตรวจจับทันทีเมื่อมีข้อมูลผิดประเภทในตาราง เช่น แอดมินระบุว่าช่อง "จำนวนเด็กตกหล่น" ต้องกรอกเป็นตัวเลขจำนวนจริง แต่ผู้ใช้งานใส่คำว่า "ไม่มี" หรือ "N/A" ลงในช่อง Excel ระบบจะปฏิเสธการแทรกลงฐานข้อมูลและไฮไลท์แถวนั้นเป็นสีแดงเพื่อเตือนให้ไปเปลี่ยนเป็นค่าตัวเลข `0` ก่อนนำเข้าใหม่
