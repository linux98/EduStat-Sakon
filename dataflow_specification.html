storage 2. โครงสร้างการจัดเก็บข้อมูลเชิงสัมพันธ์ใน Google Sheets

ระบบใช้ Google Sheets เป็นระบบจัดเก็บข้อมูลเชิงสัมพันธ์ โดยมีการเชื่อมต่อระหว่างตารางผ่านคีย์หลักและคีย์นอกเพื่อคงสถานะ Referential Integrity ดังนี้:

ตารางที่ 1: Users (สิทธิ์ผู้เข้าใช้งาน) Users Sheet
ใช้เก็บข้อมูลประจำตัวเพื่อเข้าใช้ระบบ บันทึกชื่อ สังกัด และรหัสผ่านที่ถูกเข้ารหัส SHA-256
ชื่อฟิลด์ ชนิดข้อมูล คีย์ คำอธิบาย
username String (Text) PK ชื่อเข้าใช้งานระบบ (ต้องไม่ซ้ำกัน)
password String (SHA-256) - รหัสผ่านที่เข้ารหัสลับรวม Salt ป้องกันแฮกเกอร์
role String (Enum) - สิทธิ์: super_admin, admin, agency_user
name String (Text) - ชื่อ-นามสกุลจริงของเจ้าหน้าที่
agencyId String (FK) FK รหัสสังกัดที่เชื่อมไปยัง ตาราง GisCoords / Master Data
status String (Enum) - สถานะการใช้งาน: Active, Inactive
email String (Email) - อีเมลจริงสำหรับการส่งแจ้งเตือนและทวงงาน AI
ตารางที่ 2: Settings (การตั้งค่าระบบ) Settings Sheet
เก็บค่าตัวแปรระดับโกลบอลของระบบเพื่อประมวลผลและการอ้างอิงปีการศึกษา
ชื่อฟิลด์ ชนิดข้อมูล คีย์ คำอธิบาย
key String (Text) PK ชื่อคีย์ตั้งค่า เช่น academic_year, system_status
value String (Text) - ค่าที่กำหนดให้กับคีย์นั้นๆ
ตารางที่ 3: FormTemplates (แบบฟอร์มคำถาม) Templates Sheet
เก็บเทมเพลตฟอร์มรายชื่อคอลัมน์ โดยใช้การ Serialize ข้อมูลคำถามย่อยให้อยู่ในฟิลด์เดียวเป็น JSON String ป้องกันตารางชีตแตกกระจาย
ชื่อฟิลด์ ชนิดข้อมูล คีย์ คำอธิบาย
formId String (UUID) PK รหัสระบุแบบฟอร์มเฉพาะ
formName String (Text) - ชื่อเรื่อง/หัวข้อแบบรายงานเก็บข้อมูล
agencyId String (Text) - รายชื่อสังกัดที่ต้องส่ง (คั่นด้วยเครื่องหมายจุลภาค `,`)
deadline String (Date) - วันที่กำหนดส่งสุดท้ายของแบบรายงานนี้
formConfig JSON (String) - โครงสร้างฟิลด์ ย่อย ประเภทข้อมูล ตัวอย่างคำตอบ กราฟิกการ์ด
ตารางที่ 4: Data (ผลนำส่งรายงานของหน่วยงาน) Data Sheet
บันทึกคำตอบจริงที่แต่ละหน่วยงานนำส่ง โดยข้อมูลคำถามหลายช่องจะถูกยุบมารวมลงในช่อง JSON String ของ `formData`
ชื่อฟิลด์ ชนิดข้อมูล คีย์ คำอธิบาย
dataId String (UUID) PK รหัสรายการคำตอบ
formId String (FK) FK รหัสแบบฟอร์มอ้างอิงจาก FormTemplates
agencyId String (FK) FK รหัสสังกัดที่นำส่งรายงาน
username String (FK) FK บัญชีผู้กรอกนำส่งข้อมูลจริง
status String (Enum) - สถานะตรวจรับ: Draft, Pending, Approved, Rejected
formData JSON (String) - ชุดข้อมูลผลลัพธ์คำตอบทั้งหมดที่กรอกนำส่ง
timestamp DateTime - วันและเวลาที่มีการส่งบันทึกเข้าสู่ระบบ
ตารางที่ 5: AuditLogs (ประวัติกิจกรรมระบบ) Logs Sheet
เก็บบันทึกประวัติกิจกรรมสำคัญเพื่อความโปร่งใส ปลอดภัย และการตรวจสอบย้อนกลับ
ชื่อฟิลด์ ชนิดข้อมูล คีย์ คำอธิบาย
logId String (UUID) PK รหัสบันทึกกิจกรรมย่อย
timestamp DateTime - วันเวลาที่เกิดเหตุการณ์นั้นขึ้น
username String (FK) FK บัญชีผู้ใช้ที่เป็นคนสั่งกระทำการ
role String (Text) - บทบาทของผู้ปฏิบัติการขณะทำรายการ
action String (Text) - ชื่อเหตุการณ์หลัก เช่น login_success, submit_data
details String (Text) - คำอธิบายเพิ่มเติม เช่น ชื่อแบบฟอร์มที่ลบ หรือผู้ใช้ที่สร้างใหม่
ตารางที่ 6: GisCoords (พิกัดภูมิศาสตร์) GIS Sheet
เก็บตำแหน่ง GPS ละติจูดและลองจิจูดของสำนักงานแต่ละสังกัด ใช้ปักหมุดในแผนที่และวิเคราะห์ความหนาแน่น
ชื่อฟิลด์ ชนิดข้อมูล คีย์ คำอธิบาย
agencyId String (FK) PK FK รหัสสังกัดต้นสังกัดการปักหมุดแผนที่
lat Number (Float) - พิกัดละติจูด (Latitude) ทางภูมิศาสตร์
lng Number (Float) - พิกัดลองจิจูด (Longitude) ทางภูมิศาสตร์
warning
ทำไมต้องใช้ JSON Stringified: การเก็บคำตอบคำถามย่อย (ในตารางชีต Data ฟิลด์ formData) และหน้าตาคอลัมน์ของแบบฟอร์ม (ในตารางชีต FormTemplates ฟิลด์ formConfig) เป็นข้อความ JSON แทนการกระจายเป็นคอลัมน์ Excel ทั่วไป ช่วยอำนวยความสะดวกให้ผู้ดูแลระบบสามารถ แก้ไขประเภทคำถาม เพิ่มคำถาม หรือลบประเภทการกรอกข้อมูลได้ตลอดเวลาแบบ Dynamic โดยไม่ต้องตามไปสลับ ลบ หรือเพิ่มคอลัมน์ใหม่ใน Excel ดิบ ซึ่งเป็นสาเหตุหลักทำให้ฐานข้อมูลเชิงสัมพันธ์แตกและสูญหาย
