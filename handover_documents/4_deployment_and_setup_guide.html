คู่มือการติดตั้ง ดีพลอย และตั้งค่าระบบ สำหรับฝ่ายเทคโนโลยีสารสนเทศ

เอกสารฉบับนี้จัดทำขึ้นเพื่อให้เจ้าหน้าที่ควบคุมระบบคอมพิวเตอร์และทีมงานพัฒนาระบบคอมพิวเตอร์ของผู้รับจ้าง สามารถดำเนินกระบวนการตั้งค่าระบบใหม่ทั้งหมดจากศูนย์ (Clean Install) ตลอดจนการทำซ้ำ (Re-deployment) ซิงโครไนซ์โค้ด และจัดทำระบบสำรองข้อมูล

1. อุปกรณ์และเครื่องมือที่จำเป็นต้องใช้ (Prerequisites)

ก่อนเริ่มต้นติดตั้ง โปรดตรวจสอบว่าระบบของท่านพร้อมใช้งานและติดตั้งเครื่องมือดังต่อไปนี้แล้ว:

  • Node.js: แนะนำรุ่น v18.0.0 LTS หรือสูงกว่าเพื่อใช้จัดการแพ็กเกจ
  • Google Account: บัญชี Google Workspace หรือบัญชี Gmail ปกติที่ได้รับสิทธิ์เป็นเจ้าของชีตข้อมูลและโปรเจกต์สคริปต์
  • Google Apps Script API: ตรวจสอบว่าเปิดใช้งานสิทธิ์ Google Apps Script API แล้ว โดยเข้าไปที่ script.google.com/home/usersettings และเลือกสถานะเป็น "On" (เปิดใช้งาน)
2. ขั้นตอนการตั้งค่า CLASP และการดึงโค้ดสู่ระบบคอมพิวเตอร์

clasp (Command Line Apps Script Projects) คือเครื่องมือของ Google สำหรับการเขียนโค้ดบนเครื่องคอมพิวเตอร์ (Local Editor) แล้วอัปโหลดขึ้นสู่ Google Cloud Platform มีวิธีการตั้งค่าดังนี้:

ขั้นตอนที่ 2.1: ติดตั้ง Clasp แบบ Global บนเครื่องคอมพิวเตอร์

npm install -g @google/clasp

ขั้นตอนที่ 2.2: ลงชื่อเข้าใช้ผ่านเว็บเบราว์เซอร์เพื่อเชื่อมบัญชี Google

เปิด Command Prompt หรือ PowerShell แล้วรันคำสั่งด้านล่าง ระบบจะแสดงเว็บเบราว์เซอร์ให้ลงชื่อเข้าใช้ ให้เลือกบัญชี Google ที่ผูกกับระบบและยินยอมให้สิทธิ์

clasp login

ขั้นตอนที่ 2.3: การตั้งค่าโฟลเดอร์โครงการและการระบุ Script ID

ในโฟลเดอร์โปรเจกต์ ตรวจสอบว่ามีไฟล์ .clasp.json หรือไม่ หากยังไม่มีให้สร้างไฟล์ .clasp.json แล้วกำหนดค่าดังนี้:

{
  "scriptId": "AKfycbwXzHXU0JUbdMQlkCa7zkWrWN2hWbai6cCFaU8-JzygE3BYviI19-_NQPzG9396RDF-Ug",
  "rootDir": "."
}
คำแนะนำการดึง Script ID:
เข้าสู่หน้าสคริปต์ Google Apps Script เลือกเมนู Project Settings (ไอคอนรูปฟันเฟืองทางซ้าย) จากนั้นคัดลอกรหัสที่อยู่ในช่อง Script ID เพื่อนำมาใส่ในไฟล์ .clasp.json

3. การอัปโหลดโค้ดและการสร้าง Web App URL (Build & Deployment)

หลังจากเตรียมเครื่องมือและซอร์สโค้ดสำเร็จ ให้ดำเนินตามขั้นตอนดังต่อไปนี้เพื่อเผยแพร่แอพพลิเคชันขึ้นคลาวด์:

ขั้นตอนที่ 3.1: อัปโหลดโค้ดขึ้น Apps Script Server

ใช้คำสั่ง clasp push เพื่อส่งไฟล์ทั้งหมด (รหัส.js, Index.html, JS_App.html, JS_BI.html, ReportBuilder.html, CSS.html, appsscript.json) ขึ้นคลาวด์

clasp push

ขั้นตอนที่ 3.2: การ Deploy และรับ Web App URL

หากเป็นการ Deploy ครั้งแรก แนะนำให้ทำผ่านบราวด์เซอร์ Apps Script Editor โดยทำดังนี้:

  1. ที่หัวข้อขวาบน กดปุ่ม Deploy (การพัฒนาใช้งาน) > เลือก New deployment
  2. ตั้งค่า Select type เป็น Web app (แอปพลิเคชันเว็บ)
  3. กำหนดคำตั้งค่าดังนี้:
    • Execute as: เลือก Me (อีเมลผู้ตั้งค่าระบบ) — เพื่อให้โปรแกรมทำงานภายใต้สิทธิ์บัญชีสารสนเทศกลางและเขียนลง Google Sheets ได้
    • Who has access: เลือก Anyone (ทุกคน) — เพื่อเปิดให้ผู้รายงานของทุกสังกัดและโรงเรียนเข้ากรอกข้อมูลได้โดยไม่ต้องเข้าระบบ Google
  4. คลิกปุ่ม Deploy จากนั้นคัดลอก Web App URL ไปใช้งาน
4. การตั้งค่า Properties และ API Key ของระบบ

ระบบใช้ความสามารถของ Gemini 1.5 Flash ในการประมวลผลข้อมูลและทวงงาน ซึ่งต้องใช้ API Key โดยตั้งค่าดังนี้:

  1. ไปที่หน้าโครงการ Apps Script Editor คลิกที่ไอคอน Project Settings (รูปฟันเฟือง)
  2. เลื่อนลงมาด้านล่างสุดในหัวข้อ Script Properties (คุณสมบัติของสคริปต์)
  3. คลิกปุ่ม Add script property แล้วเพิ่มค่าคู่ตัวแปรดังนี้:
    • Property (คุณสมบัติ): GEMINI_API_KEY
    • Value (ค่า): (ใส่ API Key ที่ได้รับจากเว็บ aistudio.google.com)
  4. กดปุ่ม Save script properties เพื่อบันทึกข้อมูล
5. การเปิดใช้งานตัวส่งเมลทวงถามประจำวัน (Daily Trigger Setup)

ฟีเจอร์ AI Auto-Dunning จะทำงานส่งอีเมลแจ้งเตือนการค้างส่งข้อมูลให้แต่ละโรงเรียนแบบอัตโนมัติทุกเช้า โดยต้องการทริกเกอร์ในการรันคำสั่ง วิธีตั้งค่าทำได้ 2 รูปแบบ:

วิธีที่ 1: ตั้งค่าผ่านสคริปต์คำสั่งกลาง (แนะนำ)

ให้เข้าไปที่หน้า Apps Script Editor เลือกฟังก์ชัน setupDailyTrigger ในแถบตัวเลือกด้านบน จากนั้นคลิกปุ่ม Run ระบบจะทำการตรวจสอบและสร้าง Trigger ขึ้นมาให้โดยอัตโนมัติ (ตั้งให้รันฟังก์ชัน sendDeadlineReminders ทุกวัน เวลา 08:00 น.)

วิธีที่ 2: ตั้งค่าด้วยตนเอง (Manual Trigger)

ไปที่เมนู Triggers (รูปนาฬิกาซ้ายมือ) > คลิกปุ่ม Add Trigger (ขวาล่าง) > เลือกฟังก์ชัน sendDeadlineReminders > เลือกประเภท Time-driven > เลือก Day timer > เลือกช่วงเวลา 8am to 9am > กด Save

7. วิธีการแก้ไขปัญหาเบื้องต้นของระบบ (Troubleshooting)

ปัญหาที่ 7.1: เข้าใช้งานหน้าเว็บแล้วขึ้นข้อความแจ้งเตือนสีแดง หรือโหลดชาร์ตกราฟไม่ผ่าน

  • สาเหตุ: ไม่พบคุณสมบัติ GEMINI_API_KEY ในระบบ หรือใส่คีย์ของ Gemini ไม่ถูกต้อง
  • การแก้ไข: เข้าไปเช็คใน Script Properties ว่าสะกดถูกหรือไม่ และลองคัดลอกคีย์ใหม่จาก AI Studio มาวางอีกครั้ง จากนั้นรันดีพลอยใหม่เพื่อรีเฟรชค่าแคช

ปัญหาที่ 7.2: นำเข้าไฟล์ CSV สังกัดแล้วภาษาไทยแสดงผลเป็นเครื่องหมายคำถามหรืออ่านไม่ออก (Garbled Characters)

  • สาเหตุ: ไฟล์ CSV ไม่ได้บันทึกในรูปแบบ UTF-8 BOM ทำให้ระบบไม่ทราบรูปแบบการถอดรหัสของตัวอักษรไทย
  • การแก้ไข: แนะนำให้เจ้าหน้าที่คลิกดาวน์โหลดที่ปุ่ม "เทมเพลต CSV" ภายในแอปพลิเคชันโดยตรง เนื่องจากระบบได้ฝังรหัส UTF-8 BOM ไว้ที่ส่วนหัวไฟล์แล้ว หากแก้ไขใน Excel ให้คลิก Save As > เลือกประเภทเป็น CSV UTF-8 (Comma delimited) (*.csv)

ปัญหาที่ 7.3: ข้อมูลมีปัญหาเขียนทับกันเมื่อผู้ใช้รายงานเข้ามาพร้อมกันจำนวนมาก (Concurrency Conflict)

  • สาเหตุ: เกิดจากปัญหาเขียนตารางทับกันในการดึงเลข ID ล่าสุด
  • การแก้ไข: ในซอร์สโค้ดฟังก์ชัน saveData และ registerUser ได้ติดตั้ง LockService.getScriptLock() ไว้เพื่อบล็อคคิวการเขียนชีตเรียงแถวแล้ว ตรวจสอบว่าในไฟล์ รหัส.js ไม่ได้มีการไปลบคำสั่ง lock.waitLock(10000) ออก
