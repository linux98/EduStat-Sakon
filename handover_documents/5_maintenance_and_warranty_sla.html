shield เอกสารหมายเลข 5: แผนการบำรุงรักษาและการรับประกันผลงาน — SLA
แผนการบำรุงรักษาและการรับประกันผลงาน — EduData Sakon หน้า 1
แผนการบำรุงรักษา การรับประกันผลงาน และการกู้คืนระบบคอมพิวเตอร์

ระบบจัดเก็บสถิติเชิงลึกจังหวัดสกลนคร (EduData Sakon) อยู่ภายใต้การรับประกันความเสียหายและการบำรุงรักษา เพื่อให้มั่นใจว่าหน่วยงานผู้ว่าจ้างจะได้รับระบบที่มีเสถียรภาพ ทำงานได้รวดเร็ว และมีขั้นตอนการรับมือภัยพิบัติที่เป็นระบบสากล

1. ขอบข่ายการรับประกันผลงานโครงการ (System Warranty)

ระยะเวลาการรับประกัน: ๑ ปี นับถัดจากวันที่คณะกรรมการตรวจรับพัสดุได้ลงนามอนุมัติรับรองผลการรับมอบงานจ้างงวดสุดท้ายเรียบร้อยแล้ว (เริ่มตั้งแต่วันที่ ๕ มิถุนายน ๒๕๖๙ ถึงวันที่ ๔ มิถุนายน ๒๕๗๐)

ขอบเขตความรับผิดชอบของผู้รับจ้างในระยะเวลาประกัน:

  • การตรวจสอบแก้ไขข้อบกพร่องของระบบงานจ้าง (Bug Fixing) ที่เกิดจากโครงสร้างซอร์สโค้ดเดิม หรือการทำงานที่ผิดพลาดของโปรแกรม ให้กลับมาทำงานได้ดีเป็นปกติ
  • การดูแลให้ระบบสามารถเชื่อมต่อกับ Gemini AI API และ Leaflet Map ได้อย่างราบรื่นโดยไม่เกิดข้อผิดพลาด 500 หรือ Timeout
  • ให้บริการคำแนะนำ คำปรึกษา และตอบคำถามทางโทรศัพท์หรือช่องทางออนไลน์แก่เจ้าหน้าที่คอมพิวเตอร์และทีมแอดมินของผู้ว่าจ้างในวันและเวลาราชการ
2. แผนการสำรองข้อมูลเชิงรุก (Data Backup Protocol)

เนื่องจากระบบจัดเก็บข้อมูลลงใน Google Sheets เป็นหลัก การสำรองข้อมูลจึงประกอบด้วยการดำเนินงานแบบคลาวด์และแบบแมนนวลดังนี้:

2.1 การสำรองข้อมูลอัตโนมัติด้วยระบบ Google Sheets Version History

ระบบ Google Sheets จะจัดทำประวัติการแก้ไขข้อมูลโดยละเอียดทุกวินาทีโดยอัตโนมัติอยู่แล้ว แอดมินสามารถเปิดดู ย้อนกลับ หรือดาวน์โหลดไฟล์รุ่นประวัติย้อนหลังได้ตลอดเวลาผ่านเมนู File > Version history > See version history

2.2 การสำรองข้อมูลสำเนาแผ่นงานรายเดือน (Monthly Manual Backup)

ทุกวันที่ ๑ ของเดือน ให้เจ้าหน้าที่สารสนเทศทำการดาวน์โหลดฐานข้อมูลหลัก Google Sheets ออกมาเป็นไฟล์ Excel (.xlsx) หรือสำเนาชีตเก็บไว้ใน Google Drive อีกโฟลเดอร์หนึ่งที่ใช้ชื่อว่า EduData_Backup_YYYYMMDD เพื่อป้องกันกรณีผู้ใช้งานเผลอลบแผ่นงานชีต

3. แผนการกู้คืนระบบจากภัยพิบัติ (Disaster Recovery Plan)

ในกรณีที่แผ่นงานฐานข้อมูลถูกลบหรือโครงสร้างแถวข้อมูลเสียหายรุนแรง ให้ปฏิบัติตามแผนกู้คืนระบบดังต่อไปนี้:

  1. วิเคราะห์และระบุความเสียหาย: ตรวจสอบประวัติการแก้ไขของชีตข้อมูลเพื่อค้นหาผู้ใช้ที่เป็นคนแก้ไขล่าสุด
  2. กู้คืนผ่านระบบ Version History: หากสามารถกู้ได้ ให้คลิกเลือกประวัติในระบบในช่วงเวลาก่อนหน้าที่จะเกิดความเสียหาย แล้วกดปุ่ม Restore this version
  3. การนำข้อมูลจาก AuditLogs มาย้อนกระบวนการ: ในกรณีที่ฐานข้อมูลชีต Data แถวขาดหายบางส่วนและไม่ต้องการกู้คืนทั้งชีต (เนื่องจากอาจทำให้รายการของผู้อื่นหายด้วย) ให้แอดมินเข้าไปเปิดที่ชีต AuditLogs ดึงประวัติกิจกรรม ADD_DATA หรือ EDIT_DATA ซึ่งจะมีข้อมูลชุดดิบที่ส่งเข้ามาบันทึกสำรองในช่องรายละเอียด (Detail) จากนั้นให้นำข้อมูลนั้นไปคีย์นำเข้าใหม่อีกครั้ง
  4. กรณีลิงก์สคริปต์ขาดหาย: หากไฟล์ระบบ Web App ไม่สามารถติดต่อฐานข้อมูลได้ ให้ตรวจสอบสิทธิ์การแชร์ชีตว่าถูกตั้งเป็น "Restricted" (จำกัดผู้เข้าถึง) หรือไม่ ให้แก้สิทธิ์กลับเป็นผู้ถือกรรมสิทธิ์ร่วมกับผู้ดีพลอยสคริปต์
4. ข้อตกลงระดับบริการการตอบสนองปัญหา (Service Level Agreement: SLA)

เพื่อรับประกันประสิทธิภาพการสนับสนุนหลังการส่งมอบ โครงการนี้กำหนดเกณฑ์การตอบสนองและแก้ไขความผิดพลาดตามตารางระดับความรุนแรงดังต่อไปนี้:

ระดับปัญหา ลักษณะความผิดพลาดของระบบ เวลาเข้าตรวจสอบ (Response Time) เวลาการแก้ไขเสร็จ (Resolution Time) ช่องทางแจ้งปัญหา
ระดับ 1 (Critical) ระบบล่มทั้งหมด (Web App Down), ฐานข้อมูลเสียหายไม่สามารถใช้งานได้ หรือเข้าใช้งานไม่ได้เลย 100% ภายใน 1 ชั่วโมง ภายใน 4 ชั่วโมง โทรศัพท์สายด่วน / กลุ่มแชท Line ด่วน
ระดับ 2 (Major) ระบบบางเมนูใช้การไม่ได้ เช่น AI Chatbot ไม่วาดกราฟ, แผนที่ GIS หมุดไม่ขึ้น หรือไม่สามารถนำเข้า CSV ได้ ภายใน 4 ชั่วโมง ภายใน 24 ชั่วโมง ระบบตั๋วแจ้งปัญหา / กลุ่มแชท Line ไอที
ระดับ 3 (Minor) ความผิดพลาดด้านสีสันหน้าจอ, คำสะกดผิดบนหน้าเว็บเมนู หรือปัญหาเล็กน้อยที่ไม่กระทบการกรอกส่งสถิติ ภายใน 24 ชั่วโมง ภายใน 72 ชั่วโมง อีเมลแจ้งปัญหา / กลุ่มแชท Line ไอที
